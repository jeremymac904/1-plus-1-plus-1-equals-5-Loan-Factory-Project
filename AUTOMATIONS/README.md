# AUTOMATIONS

Light-automation requests, runbooks, and planning for Phase 2 builds.

## What goes here

- Automation requests (use `TEMPLATE_AUTOMATION_REQUEST.md`)
- Automation runbooks once shipped
- Phase 2 automation backlog (parked items)
- Zapier / Google Forms / Chat alert configs

## Scope rule

**Light automation (in scope for Phase 1):**

- Google Form → Chat alert
- Google Form → intake ticket
- Existing platform reporting → Google Sheet rollup
- Email Drip sequences in the portal
- Ally posting consistency tracking (already exists)

**Real engineering work (Phase 2 only):**

- Custom dashboards
- AI personalization pipelines
- Microsite generators
- Cross-platform integrations beyond what existing systems support

Don't propose Phase 2 builds for June 1 unless the pilot proves they're mandatory.

## Naming

- Requests: `YYYY-MM-DD_short_title_automation_request.md`
- Runbooks: `descriptive_name_runbook.md`
- Phase 2 backlog: `phase_2_backlog.md` (single file with running list)

Use `TEMPLATE_AUTOMATION_REQUEST.md`.
