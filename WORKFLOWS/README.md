# WORKFLOWS

Operational workflows that define how things happen end-to-end.

## What goes here

- Team Leader onboarding workflow
- Content publishing workflow (per channel)
- Compliance review workflow
- Pilot Leader meeting run-of-show
- Escalation workflow
- 30-day team support workflow
- Cross-team content syndication workflow (post-launch)

## What doesn't

- One-off process changes (those go in `MEETING_NOTES/` if discussed in a meeting)
- Strategy documents (those are root-level)

## Naming

`descriptive_workflow_name.md` — all lowercase, underscores.

Example: `team_leader_onboarding.md`, `compliance_review_template_level.md`

## Template

Use `TEMPLATE_WORKFLOW_DOC.md` — copy, rename, fill in.

## When workflows change

1. Update the workflow doc in place
2. Note the change in the next Pilot Leader meeting recap
3. Update any docs that cross-reference it
4. If the change supersedes a previous version, move the old version to `ARCHIVE/`
