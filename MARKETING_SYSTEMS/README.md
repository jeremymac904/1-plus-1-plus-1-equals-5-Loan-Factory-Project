# MARKETING_SYSTEMS

Marketing-owned artifacts: the team kit, templates, brand variants, and Marketing pod request log.

## What goes here

- Team kit v1 (and subsequent versions) — logo lockups, color palette, typography rules
- Template library (GBP post, newsletter, social post, Realtor outreach, podcast intro/outro)
- Brand variants per division (Spanish, Vietnamese, Russian, Korean, etc.)
- Compliance language library
- Marketing requests log
- Marketing pod's first-30-day support plan templates

## What doesn't

- Customer-facing published content (that gets published from Ally / portal / GBP, not stored here)
- Platform setup details (those go in `TERA_AND_TECH_REQUIREMENTS/`)
- AI drafting prompts (those go in `PROMPTS/`)

## Naming

- Team kit assets: `team_kit_v[N]_[component].[ext]`
- Templates: `template_[channel]_[variant].md` or `.html`
- Requests: `YYYY-MM-DD_short_title_marketing_request.md`

## Template

Use `TEMPLATE_MARKETING_REQUEST.md` for new requests to the Marketing pod.

## Brand kit source

Logo files, full-resolution brand assets, and design source files live in Marketing's working folder in Drive. Reference paths here. Don't commit large binaries.

## Cross-references

- `PROMPTS/` — AI drafting prompt library
- `TEAM_LEADER_PROGRAM/` — team profiles that consume marketing assets
- `LANGUAGE_DIVISIONS/` — division-level brand customization
