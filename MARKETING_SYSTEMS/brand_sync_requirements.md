# Brand Sync Requirements

**Owner:** Marketing pod lead (production), Jeremy McDonald (operational), TERA pod (technical surfaces), Compliance (template review)
**Status:** Direction documented, Wave 1 enforced manually via the team kit
**Last updated:** 2026-05-18
**Purpose:** Lock the principle that the same branding and content structure carries across every team-facing surface so a borrower, realtor, or recruit sees one consistent team across website, email, social, GBP, outreach, video, podcast, and reporting.

## The principle

Same team, same brand, every surface. A borrower who finds a team on Google Business Profile, then opens its newsletter, then sees its Instagram post, then visits its website should see the **same logo, palette, voice, disclosures, and structure**. Each surface adapts to format. None of them invents a new identity.

## Surfaces in scope

The team kit and content structure must carry across:

1. **Team website pages** (Loan Factory website team pages)
2. **Individual LO profile or landing pages** where applicable (each LO inherits the team brand with personal layering)
3. **Email newsletters** (Loan Factory portal newsletter audience)
4. **Email drip campaigns** (Loan Factory portal Email Drip)
5. **Social media content inside ALLY** (Facebook, Instagram, LinkedIn personal, LinkedIn company, YouTube, GBP if syndicated, TikTok if applicable, X if applicable, Threads if applicable)
6. **Google Business Profile posts** (per market)
7. **Realtor outreach campaigns** (email + 1:1 outreach kits)
8. **YouTube and podcast content** if the team opts in
9. **Internal scorecards and reporting** (Team Leader scorecard, launch tracking, dashboards)

If a future surface is added (e.g., paid ads, microsites, SMS), it must conform to the same kit. No exceptions without a kit version bump.

## What "same brand" means

| Element | Standard |
| --- | --- |
| Logo lockup | One primary, one secondary, one icon mark. Approved set in the kit. |
| Color palette | Primary, secondary, accent, neutral, alert. Hex values locked. |
| Typography | Headline font, body font, mono if used. Web-safe fallbacks defined. |
| Voice and tone | Defined per division and per language. Same across every surface. |
| Tagline | One tagline per team, optional division-specific variant. |
| Imagery direction | Approved photography style, illustration style if any, do-not-use list. |
| Disclosures | NMLS, EHL, state-specific, trigger term, $2,000 Best Price Guarantee. Baked into every template per D9. |
| Required structure | Header, body, CTA, disclosure block. Consistent placement across surfaces. |

## What "same content structure" means

Content structure is the order and presence of standard elements per format. Each surface uses the same structure adapted to that format:

| Format | Required structure |
| --- | --- |
| Newsletter | Header → Team intro → Section 1 (market or rate update) → Section 2 (educational) → Section 3 (community or team) → CTA → Disclosures |
| Drip email | Subject line → Hook → Value → Soft CTA → Disclosures |
| Social post | Hook → Value → CTA → Disclosure (where applicable) |
| GBP post | Hook → Value → CTA → Photo + branded asset |
| Realtor outreach | Personalized opener → Team intro → Value to their book → CTA → Disclosure |
| Video script | Hook (3 sec) → Value (30 sec) → CTA → Disclosure overlay |
| Podcast episode | Intro music → Episode hook → Body → Recap → Outro CTA → Disclosure read |
| Website team page | Hero → Team intro → LO grid → Languages and markets → Programs → CTA → Disclosures footer |
| LO landing page | Hero (LO photo) → Bio → Languages → Markets → Programs → Reviews → CTA → Disclosures footer |

## Sync workflow

1. Master record locked → Marketing builds team kit v1
2. Compliance reviews kit templates for required disclosures (template-level, per D9)
3. Marketing distributes kit assets to TERA for every technical surface
4. TERA applies kit to website, LO pages, newsletter audience, drip sequence, ALLY config, GBP, realtor outreach tool stack
5. Team Leader approves the first content batch on each surface
6. Bi-weekly Pilot Leader meeting surfaces any drift; Marketing reissues affected templates if needed
7. Kit version bumps are versioned (`team_kit_v1`, `team_kit_v2`, etc.) and rolled across every surface together, never piecemeal

## Drift prevention

A drift is when one surface starts publishing in a different brand than the others. Detection points:

- Pilot Leader meeting (qualitative)
- Launch tracking sheet column "Brand consistency check" (weekly)
- Spot audit by Marketing pod rep during first-30-day support
- Compliance escalation if a disclosure goes missing on any surface

Drift response:

1. Surface owner pauses publishing on the drifting surface
2. Marketing reissues the affected template
3. TERA applies the corrected template
4. Logged in launch tracking + flagged in next Pilot Leader meeting
5. Repeated drift triggers kit version bump and full re-deploy

## Language and division layering

Each language and each division gets an approved variant of the kit. The variant inherits the parent team brand and adds:

- Language-specific typography (e.g., Vietnamese diacritics)
- Locally translated tagline (Marketing translation, not machine)
- Culturally appropriate imagery direction
- Division-specific tagline if applicable (e.g., Veteran division variant)

A team operating in three languages publishes the same core content in three branded variants, not three different brands.

## Reporting tie-in

Brand consistency is a tracked field on the launch tracking sheet. The Marketing rep assigned to each team checks weekly that every surface is on-brand. The Team Leader scorecard reflects brand adherence as a soft KPI.
